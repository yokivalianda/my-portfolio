Panduan Setup Firebase

Ikuti langkah berikut — tidak perlu coding! Hanya butuh ±15 menit ☕

Yang Anda Butuhkan

Akun Google (Gmail)
Browser (Chrome/Firefox/Edge)
File portfolio yang sudah diunduh
Akses ke folder file portfolio Anda
~15 menit waktu
Struktur File Portfolio Anda

Pastikan file-file ini ada di folder portfolio Anda:

📁 my-portfolio/ ├── index.html ← halaman utama (sudah diperbarui) ├── admin.html ← dashboard admin (sudah diperbarui) ├── 📁 js/ │ └── firebase-config.js ← ⭐ ANDA AKAN EDIT FILE INI ├── 📁 img/ ← gambar & foto ├── 📁 lib/ ← library lightbox dll └── PANDUAN_SETUP.html ← file panduan ini
1
Buat Project Firebase

Buka browser, kunjungi: console.firebase.google.com

Masuk dengan akun Google Anda, lalu:

  Klik tombol "Add project" atau "Tambahkan project"

Isi form pembuatan project:

  Nama project: isi bebas, contoh: portfolio-yoki
  Google Analytics: boleh dinonaktifkan (klik toggle, lalu klik Continue)
  Klik "Create project" — tunggu sebentar sampai siap
  Klik "Continue" — Anda masuk ke dashboard Firebase
2
Aktifkan Firestore Database

Di sidebar kiri Firebase Console, cari dan klik "Firestore Database"

  Jika tidak terlihat, klik ikon hamburger ☰ kiri atas → Build → Firestore Database

Lalu:

  Klik "Create database"
  Pilih "Start in production mode" → klik Next
  Pilih lokasi server: pilih "asia-southeast2" (Jakarta) untuk performa terbaik di Indonesia
  Klik "Done" — tunggu Firestore siap (biasanya 30 detik)
  Jangan khawatir dengan "production mode" — kita akan atur security rules di Langkah 5.
3
Aktifkan Authentication & Buat Akun Admin

Di sidebar kiri, klik "Authentication" (atau Build → Authentication)

  Klik tab "Sign-in method"
  Klik "Email/Password"
  Toggle "Enable" → klik Save

Sekarang buat akun admin Anda:

  Klik tab "Users"
  Klik "Add user"
  Isi Email (bisa email Gmail Anda) dan Password pilihan Anda
  Klik "Add user"
  Catat email dan password ini — Anda akan memakainya untuk login ke halaman admin!
4
Ambil Konfigurasi Firebase & Isi ke File

Di sidebar kiri, klik ikon ⚙️ Project Settings (roda gigi di pojok kiri bawah), lalu scroll ke bawah ke bagian "Your apps".

  Klik ikon </> (Web app)
  Isi nama app: contoh portfolio-web
  Klik "Register app"
  Anda akan melihat blok kode seperti ini:
// Firebase SDK configuration const firebaseConfig = { apiKey: "AIzaSyXXXXXXXXXXXXXX", authDomain: "portfolio-yoki.firebaseapp.com", projectId: "portfolio-yoki", storageBucket: "portfolio-yoki.appspot.com", messagingSenderId: "123456789012", appId: "1:123456789012:web:abcdef123456" };

Sekarang buka file js/firebase-config.js dengan Notepad / VS Code, dan isi nilainya:

// js/firebase-config.js const FIREBASE_CONFIG = { apiKey: "GANTI_dengan_apiKey_dari_Firebase", authDomain: "GANTI_dengan_authDomain", projectId: "GANTI_dengan_projectId", storageBucket: "GANTI_dengan_storageBucket", messagingSenderId: "GANTI_dengan_messagingSenderId", appId: "GANTI_dengan_appId" }; // Email yang Anda daftarkan di Step 3 const ADMIN_EMAIL = "email-admin-anda@gmail.com";
  Salin nilai persis sesuai aslinya termasuk tanda kutip dan koma. Jangan ada typo!
5
Atur Security Rules Firestore

Di Firebase Console, buka Firestore Database → klik tab "Rules"

Hapus semua teks yang ada, lalu copy-paste rules di bawah ini:

// Rules: publik bisa baca, hanya user login yang bisa tulis rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /portfolio/{document} { // Siapapun bisa baca (untuk tampilkan portfolio) allow read: if true; // Hanya user yang sudah login yang bisa edit (admin) allow write: if request.auth != null; } } }
  Paste rules di atas ke editor
  Klik tombol "Publish"
  Rules ini aman: pengunjung portfolio hanya bisa membaca data. Hanya Anda (yang sudah login) yang bisa mengubah data.
6
Test & Upload ke Hosting

Test secara lokal:

  Buka file admin.html di browser
  Login dengan email & password yang Anda buat di Step 3
  Edit konten → klik "Simpan ke Firestore"
  Buka index.html → data harus muncul dari Firestore ✅

Upload ke hosting (pilih salah satu):

Firebase Hosting (Gratis)

Hosting langsung dari Firebase. Gratis. Cepat. Cocok karena sudah pakai Firebase.

→ Lihat cara setup
Hosting Indonesia

Niagahoster / IDWebhost: upload semua file via File Manager di cPanel. Tidak perlu setting tambahan.

  Setelah upload, buka admin.html di hosting → simpan data → portfolio langsung update! Tidak perlu upload ulang.
Masalah Umum & Solusinya
❌ "Gagal: Periksa konfigurasi"
→ Buka js/firebase-config.js, pastikan semua nilai sudah diisi dengan benar. Tidak ada tulisan "GANTI_...".
❌ Login gagal "Email tidak ditemukan"
→ Pastikan email yang Anda masukkan sama persis dengan yang didaftarkan di Firebase Authentication Step 3.
❌ "Gagal menyimpan: Missing or insufficient permissions"
→ Security Rules belum dipublish. Ulangi Langkah 5 dan pastikan klik tombol Publish.
❌ Portfolio tidak muncul data baru setelah simpan
→ Buka portfolio lagi (refresh/hard refresh dengan Ctrl+Shift+R). Badge "Firebase Terhubung ✓" harus berwarna hijau.
